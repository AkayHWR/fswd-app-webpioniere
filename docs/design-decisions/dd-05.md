# 05: Allow voting on questions in addition to answers

## Meta
Status : **Decided**
Updated : 18-Jun-2026

## Problem Statement
The platform plans to support upvotes and downvotes so that the community can
give feedback on contributions. We need to decide if voting should apply to
questions as well as answers, or if it stays restricted to answers only.

## Decision

*Decision was taken by:* github.com/MasinHassan

## Regarded options

### Option 1: Allow voting on questions and answers
With this option users can vote on both questions and answers.

Pros:
- Good questions can get recognised by the community even if they don't have many answers yet. Right now there is no way to tell if a question is well written or useful without looking at the answer count, which is not really the same thing.
- It also feels more consistent from a user perspective because the same interaction is available on all content, not just on answers.
- The database already has a `question_vote` table and `upvotes` and `downvotes` columns on the question table, so we would not need to change the schema.

Cons:
- We need to implement and maintain two separate vote flows instead of one, which means more code and more places where something can go wrong.
- It also needs to be decided how question votes fit into the leaderboard once that is implemented, otherwise we have a visible mechanic that does not actually affect anything.

### Option 2: Restrict voting to answers only
With this option only answers can be upvoted or downvoted.

Pros:
- Only one vote flow needs to be implemented and maintained, which keeps things simpler especially since voting is not built yet and we want to avoid unnecessary complexity early on.
- The question detail page stays cleaner because vote buttons only appear on answer cards.

Cons:
- There is no way for users to give feedback on questions themselves. A really good or a really bad question looks the same as an average one.
- The `question_vote` table and the columns on the question table would go unused unless we clean them up in a separate step.
