# 08: Voting Permissions: Authenticated vs. Anonymous Voting

## Meta
Status : **Decided**
Updated : 21-Jun-2026

## Problem Statement
StudySwap relies on a community-driven voting system to rank high-quality answers and calculate the user leaderboard. We had to decide whether to allow anonymous users (guests) to cast votes to maximize user engagement, or to strictly require user authentication (login) prior to voting to ensure the integrity of the platform's gamification system.

## Decision
We decided to go with Option 1 (Authenticated Voting Only).
*Decision was taken by:* github.com/KarlTebbe

## Regarded options

### Option 1: Authenticated Voting (Login Required)
With this option, the voting interface is either hidden or triggers a redirect to the login page if the user does not have an active session. Votes are strictly tied to a `user_id` in the database.

Pros:
- Leaderboard Integrity: It guarantees the "One User, One Vote" principle. It is structurally impossible for a single student to artificially inflate their own score or sabotage others to climb the leaderboard.
- Accountability: Because votes are tied to accounts, the system can track voting history, allowing users to retract or change their votes later.
- Community Building: It creates a strong incentive for silent readers ("lurkers") to finally register an account if they want to participate in rewarding good content.

Cons:
- High Friction: It creates a barrier to entry. Occasional visitors who just want to quickly say "thanks" via an upvote are blocked, potentially lowering the total volume of votes across the platform.

### Option 2: Anonymous Voting 
With this option, anyone visiting the platform can click the vote buttons. The system attempts to prevent duplicate voting using browser cookies or IP address logging instead of user accounts.

Pros:
- Maximum Engagement: Zero friction means significantly more votes are cast, making the platform look highly active very quickly.
- Immediate Gratification: Guests can instantly interact with the content.

Cons:
- Vulnerable to Manipulation: Cookies can be cleared in seconds, and IP addresses change. Malicious users could write simple scripts to easily manipulate answer rankings and ruin the fairness of the leaderboard.
- Complex Backend Logic: Relying on IP tracking for voting introduces massive GDPR concerns and highly complex, error-prone database structures compared to a simple `user_id` foreign key.
