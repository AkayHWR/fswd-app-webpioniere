# 06: Restrict Solution Marking to the Question Author

## Meta
Status : **Decided**
Updated : 19-Jun-2026

## Problem Statement
StudySwap allows users to post answers to existing questions. To highlight
the best answer, we need a "Beste Antwort" mechanic. The question is who
should be allowed to mark an answer as the solution: only the person who
originally asked the question, or any registered user.

## Decision
We decided that only the original author of a question can mark an answer as
the solution.

The person who asks a question is the one with the actual problem, so they are
in the best position to judge which answer actually helped them. Letting other
users mark solutions would open the door for incorrect or even malicious
markings, which would hurt the reliability of the application. It also keeps
the implementation straightforward and matches how platforms like Stack Overflow
handle this.

*Decision was taken by:* github.com/MasinHassan

## Open Point
It is currently undecided whether future moderator roles should be able to
mark solutions for abandoned questions where the original author never came
back. This will only be relevant if we add a moderator system later.

## Regarded options

### Option 1: Author-only solution marking
With this option only the user who asked the question can accept an answer
as the solution.

Pros:
- The person with the problem knows best whether an answer actually solved it,
  which makes the accepted solution more trustworthy.
- Other users cannot mark wrong or misleading answers as correct, which keeps
  the content reliable.
- The backend check is a simple comparison of two user IDs, so the
  implementation stays clean and easy to follow.

Cons:
- If the original author never logs back in, the question stays unsolved
  permanently even if a perfect answer already exists.

### Option 2: Community-based solution marking
With this option any registered user, or users above a certain reputation
threshold, can mark an answer as the solution.

Pros:
- Open questions can be closed by the community even if the original author
  is no longer active.

Cons:
- In a university context a majority of students might agree on an answer that
  sounds right but is actually wrong, which could mislead others with the same
  problem.
- Building a proper consensus mechanic would add significant complexity to the
  project for a feature that may rarely be needed.
