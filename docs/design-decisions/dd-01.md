# 01: Use Bootstrap for the user interface or plain HTML

## Problem Statement

Our application needs several user-facing pages, including the dashboard, login, registration, question detail pages and forms for creating questions and answers. These pages should look consistent, be readable on different screen sizes and remain easy to maintain during the project.

We had to decide whether to build the interface only with custom HTML and CSS or use Bootstrap as a frontend framework on top of our HTML templates.

## Regarded options

### Option 1: Bootstrap

With this option, the application uses Bootstrap classes and components inside the HTML templates.

Pros:

- Pages can be styled consistently with less custom CSS.
- Forms, buttons and navigation elements are quicker to build.
- Responsive layouts are easier to implement.
- Bootstrap is widely used and well documented.
- Team members can understand and reuse common Bootstrap classes across templates.

Cons:

- The design may look more generic if it is not customized.
- The project depends on an additional frontend framework.
- Some custom styling is still needed for a more individual look.

### Option 2: Custom HTML and CSS only

With this option, all layouts and visual styles would be written manually without Bootstrap.

Pros:

- Full control over every visual detail.
- No dependency on a CSS framework.
- The interface could be more individual from the beginning.

Cons:

- More time would be needed for layout, spacing, form styling and responsive behavior.
- It would be easier for pages to become visually inconsistent.
- Every team member would need to follow the same CSS conventions carefully.
- The additional styling effort would take time away from the core application features.
