# 09: Restrict Answer Creation to Logged-in Users

## Meta
Status : **Decided**
Updated : 20-Jun-2026

## Problem Statement
StudySwap allows students to answer existing questions. We had to decide if
anonymous users should be able to post answers too, or if this should require
a registered account.

## Decision
We decided that only logged-in users can create answers.

Every answer should be tied to a real user account. This makes the discussion
more trustworthy because people are responsible for what they post, and it
also prevents random spam from showing up without any way to trace it back.
Having answers linked to accounts also makes it possible to add things like
reputation points, leaderboards etc.

Users that are not logged in can still read all questions and answers, they
just can't post a new one without an account first.

*Decision was taken by:* github.com/MasinHassan

## Regarded options

### Option 1: Only logged-in users can answer
Pros:
- Every answer is linked to an actual user account.
- Less spam and misuse since users can't just post without consequences.
- Makes future features like reputation systems or user statistics possible.

Cons:
- Users need to create an account before they can contribute anything.

### Option 2: Anonymous answers
Pros:
- Lower barrier for participation since no account is needed.
- Users could contribute right away without registering first.

Cons:
- Much harder to moderate since there is no user behind the content.
- No connection between answers and users at all.
- Anonymous spam becomes a lot more likely.
