# Solution Elements

## Solution Hypothesis

Wenn Studierende eine geschlossene, einfache Web App fuer erfahrungsbasierte
Studienfragen nutzen koennen, dann werden hilfreiche Antworten besser
auffindbar, Fragen weniger oft doppelt gestellt und gute Beitraege in der
Community sichtbarer.

## App Elements

| Problem aus der Discovery | Solution Element in StudySwap | Nutzen |
| --- | --- | --- |
| Wissen ist in Chats verstreut. | Zentrales Dashboard mit allen Fragen | Studierende sehen bestehende Fragen an einem Ort. |
| Fragen sind spaeter schwer zu finden. | Titel, Beschreibung und freie Hashtags | Inhalte koennen thematisch sortiert und gefiltert werden. |
| Nicht jede Antwort ist gleich hilfreich. | Upvotes und Downvotes auf Fragen und Antworten | Die Community macht hilfreiche Inhalte sichtbarer. |
| Fragesteller brauchen eine klare Loesung. | Antwort als Loesung markieren | Andere erkennen schnell, welche Antwort das Problem geloest hat. |
| Zugang soll vertrauenswuerdig bleiben. | Registrierung nur mit HWR-Mail | Die Plattform bleibt auf HWR-Studierende ausgerichtet. |
| Helfende Nutzer sollen Motivation bekommen. | Leaderboard mit Punkten | Gute Beitraege werden sichtbar anerkannt. |

## MVP Scope

Der MVP konzentriert sich auf die Kernfunktion der Value Proposition:
Studierende stellen Fragen, andere Studierende antworten, und die Community
hilft dabei, die besten Antworten sichtbar zu machen.

Umgesetzt sind dafuer:

- Login und Registrierung mit HWR-Mail
- Dashboard mit Fragen und Hashtag-Filtern
- Frage erstellen
- Detailansicht mit Antworten
- Votes auf Fragen und Antworten
- Loesung markieren
- Leaderboard
- JSON-API fuer Fragen

## Nicht im MVP

Einige Ideen passen zur langfristigen Vision, wurden aber bewusst nicht in den
ersten Scope aufgenommen:

- automatische KI-Zusammenfassungen
- Direktnachrichten zwischen Studierenden
- Upload-System fuer Dokumente
- Moderationsrollen fuer Fachschaft oder Lehrende
- Benachrichtigungen per E-Mail
