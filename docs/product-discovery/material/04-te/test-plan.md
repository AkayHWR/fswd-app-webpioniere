# Testplan

## Ziel

Der Test soll zeigen, ob StudySwap als App-Loesung fuer die Value Proposition
verstaendlich ist:

- Problem: Erfahrungswissen ist verstreut und schwer auffindbar.
- Nutzer: HWR-Studierende mit Studienfragen oder Erfahrung.
- Loesung: geschlossene Frage-Antwort-App mit Hashtags, Votes und Loesungen.

## Testbare Hypothesen

| Hypothese | Methode | Erfolgskriterium |
| --- | --- | --- |
| Nutzer verstehen den Zweck der App in unter einer Minute. | 5-Second-Test mit Dashboard oder Startseite | Testperson nennt "Fragen und Antworten fuer Studierende" sinngemaess. |
| Nutzer koennen eine relevante Frage finden. | Task: Frage zu `#Klausur` suchen | Testperson findet eine passende Frage ohne Hilfe. |
| Nutzer koennen eine neue Frage erstellen. | Task: Frage mit Titel, Beschreibung und Hashtag anlegen | Frage wird erfolgreich erstellt. |
| Nutzer erkennen hilfreiche Antworten. | Task: Detailseite mit mehreren Antworten bewerten | Testperson nutzt Votes oder Loesungsmarkierung zur Orientierung. |
| HWR-Mail erhoeht Vertrauen. | Feedbackfrage nach Registrierungskonzept | Testperson nennt geschlossenen HWR-Kontext als positiv oder nachvollziehbar. |

## Ablauf fuer einen Usability-Test

1. Kurz erklaeren: "Du testest eine App fuer Studienfragen von HWR-Studierenden."
2. Testperson ohne weitere Anleitung auf das Dashboard schauen lassen.
3. Aufgabe 1: Finde eine Frage zum Thema Klausur.
4. Aufgabe 2: Erstelle eine eigene Frage mit einem passenden Hashtag.
5. Aufgabe 3: Oeffne eine Frage und entscheide, welche Antwort hilfreich ist.
6. Aufgabe 4: Sieh dir das Leaderboard an und erklaere, was es zeigt.
7. Nachfragen: Was war klar? Was war unklar? Was fehlt?

## Zu beobachtende Signale

- Wird der Unterschied zu WhatsApp oder Discord verstanden?
- Werden Hashtags als Filter erkannt?
- Ist die Loesungsmarkierung verstaendlich?
- Wirkt das Leaderboard motivierend oder ablenkend?
- Gibt es Stellen, an denen Nutzer offizielle Informationen statt Peer-Wissen erwarten?
